CRITICAL LESSONS FROM PHASES 6-9 — APPLY ALL OF THESE, DO NOT SKIP:

1. SILENT COMMIT BUG (found in Phases 3-5, fixed): NEVER use
   `async for session in get_session(): ... return result` in a Celery task
   core function — the early return kills the generator via GeneratorExit
   before the post-yield commit runs, silently discarding all writes.
   ALWAYS use session_scope() (the pattern already correct in
   asset_tasks.py, animation_tasks.py, voice_tasks.py, music_tasks.py).
   Before reporting complete, run:
   grep -n "async for session in get_session" apps/worker/tasks/video_assembly_tasks.py
   This MUST return nothing.

2. DISPATCH SIGNATURE: every dispatcher.dispatch() call must use
   celery_task=, core_coro_factory=, job_id=, queue=, task_kwargs=. Grep
   your own new code against the real TaskDispatcher signature before
   considering it done.

3. DISPATCH_MODE UNWRAPPING: dispatcher.dispatch() returns a dict
   ({"job_id":..., "task_id":..., "mode":..., "status":...}), NOT a plain
   string. Always capture it as `dispatch_result = await dispatcher.dispatch(...)`
   then extract `mode = dispatch_result["mode"]` before using mode anywhere,
   especially before passing it into a Pydantic response model's string field.

4. UUID JSON SERIALIZATION: any `req.model_dump()` on a Pydantic request
   model containing UUID fields, used as a JSON column value, MUST use
   `req.model_dump(mode="json")` — plain `.model_dump()` leaves UUID objects
   as Python objects, which crash on JSON serialization at insert time.

5. ROUTER REGISTRATION: after building the router, you MUST add it to BOTH
   the import block AND v1.include_router(...) calls in
   backend/apps/api/main.py. Confirm by actually starting the server and
