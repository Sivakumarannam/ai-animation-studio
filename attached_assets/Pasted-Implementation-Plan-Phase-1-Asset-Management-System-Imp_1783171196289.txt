Implementation Plan — Phase 1: Asset Management System
Implement a production-ready Asset Management module supporting Character, Background, Props, Animation, Audio, Music, and Sound Effects libraries with features like versioning, preview, metadata, upload, import, export, soft delete, restore, pagination, drag-and-drop upload, and bulk operations.

Proposed Changes
We will extend the existing clean architecture by adding new SQLAlchemy models, updating existing ones to support soft-deletes/metadata, generating an Alembic migration, writing repositories/services, exposing unified REST API routes, and designing a tabbed React page.

1. Database Layer
We will modify existing models to support soft delete and metadata, and define three new models for audio-based assets.

[MODIFY]

Add is_deleted column (Boolean, default False) to Background, Prop, and AnimationPreset.

5. Dependency Injection & Routing
[MODIFY]

[MODIFY] 
asset_manager.py
Implement REST endpoints for the asset libraries under /asset-manager:
POST /asset-manager/upload — upload binary assets to MinIO.
POST /asset-manager/import — create asset record from metadata.
POST /asset-manager/export — export snapshot.
DELETE /asset-manager/{asset_type}/{asset_id} — soft delete.
POST /asset-manager/{asset_type}/{asset_id}/restore — restore.
POST /asset-manager/{asset_type}/bulk-delete — bulk soft delete.
POST /asset-manager/{asset_type}/bulk-restore — bulk restore.
POST /asset-manager/{asset_type}/bulk-update — bulk update metadata/tags.
GET /asset-manager/{asset_type} — paginated search/browse with filters.
Extend versioning endpoints to support all 7 asset types.
